## 🌟 这是什么？

这是一个**将软件工程最佳实践应用于学术论文写作**的开源项目。

传统论文写作往往是「线性」的——从文献综述写到结论，一条路走到底。但在 AI 时代，我们可以做得更好：

- 🔄 **迭代式**：像软件开发一样，论文也可以逐步迭代完善
- 🤖 **AI 协作**：让 AI 阅读你的文献、参与你的讨论、帮你修订段落
- 📋 **PR 驱动**：每次修改都是一个独立的「Pull Request」，可追溯、可审核
- 📚 **知识管理**：文献库、讨论记录、修改历史，全部结构化管理

## 🏗️ 核心架构

```
Graduate-thesis/
├── 📁 Reference/          # 文献库（AI 的知识来源）
│   ├── _INDEX.md          # 文献索引
│   └── PDF-MD/            # PDF → Markdown 转换工具
│       ├── pdfs/          # 放入 PDF，双击 run.bat
│       └── output_api/    # AI 可阅读的 Markdown
│
├── 📁 Consensus/          # 学术讨论区（与学术 AI 的对话）
│   ├── _INDEX.md          # 讨论索引
│   └── *.md               # 每次讨论一个文件
│
├── 📁 PR/                 # 修改请求（论文变更的最小单元）
│   ├── _INDEX.md          # PR 索引
│   └── *.md               # 每个 PR 独立可执行
│
└── 📁 Target/             # 论文产物
    └── Draft.md           # 论文正文
```

**工作流：**

```
文献阅读 → 学术讨论 → 形成洞见 → 创建PR → 修改论文 → 循环
    ↑                                              ↓
    └──────────────── 持续迭代 ←───────────────────┘
```

## 🚀 如何使用

### 对学生：复制这个框架到你的论文项目

```bash
git clone https://github.com/yourname/Graduate-thesis.git my-thesis
```

1. **整理文献**：把 PDF 放入 `Reference/PDF-MD/pdfs/`，运行 `run.bat` 自动转换
2. **记录讨论**：与 ChatGPT/Claude 等讨论学术问题，保存到 `Consensus/`
3. **创建 PR**：每次想修改论文，先在 `PR/` 创建修改请求
4. **迭代完善**：审核 PR，合并到 `Target/Draft.md`

### 对 AI 开发者：关注 `agent.md` 文件

每个文件夹都有一个 `agent.md`，这是给 AI agent 的「使用说明」：
- 定义该模块的职责边界
- 规定 AI 的行为规范
- 提供操作模板和示例

## 💡 核心原则

| 原则 | 解释 |
|------|------|
| **讨论是中心** | 所有洞见来自与 AI 的学术对话 |
| **PR 独立性** | 每个修改请求完整独立，无需上下文 |
| **文献即真相** | Reference 是唯一论据来源，不凭空捏造 |
| **问题要深入** | 提问须具体，避免「帮我写文献综述」 |
| **循环迭代** | 讨论 → 文献 → 洞见 → PR → 讨论... |

## 🔧 PDF-MD 工具

自动将 PDF 学术文献转换为 Markdown，让 AI 能够「阅读」你的参考文献。

**特性：**
- ✅ 调用 Marker API 高质量转换
- ✅ 自动提取论文标题并重命名
- ✅ 自动更新文献索引
- ✅ 超过 50 页的文件会提示确认

**使用：**
```bash
# 1. 设置 API Key（首次）
# 在项目根目录创建 .env 文件：DATALAB_API_KEY=your_key

# 2. 放入 PDF，运行
cd Reference/PDF-MD
python process.py
# 或双击 run.bat
```
