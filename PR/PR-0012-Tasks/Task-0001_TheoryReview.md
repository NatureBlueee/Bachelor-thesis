## 用户指导原则（2025-12-29）

> **用户原文**：
> "你这个有点太局限了，太过于刻板了。每一个指标能扩展出很多的问题。我们先把问题先丰富出来，然后再根据不同的人群去做不同的问题的策略。比如假如他是学生，那么是什么样的策略？假如他是知识工作者，那可能是哪样的策略？"
>
> "问问题的方式，就是它不一定是直接问你想问的东西，而是通过侧面或者跟日常话的语言或者更精确的方式去。"
>
> "如果你需要一些最新的知识，你也可以去搜一下，一些术语，是吧？一些比如说AI圈子最近都在讨论些什么，像卡帕西啊，或者是奥特曼呢，或者是达里奥啊，他们在做些什么，他们在聊些什么？"

**设计原则**：
1. 先发散再收缩
2. 侧面/日常化语言
3. 反映真实AI语境（Karpathy, Altman, Dario, AI播客话题）
4. 分人群策略

## 当前AI圈热点话题（搜索结果 2024-12）

| 热点 | 内容 | 可用于访谈的切入点 |
|------|------|-------------------|
| **Karpathy vs Altman辩论** | Karpathy认为当前Agent是"slop"，还需10年；Altman认为2025Agent将成为"虚拟同事" | "你怎么看Agent？觉得现在能用吗？" |
| **多Agent协作** | LangChain、AutoGPT、CrewAI等多Agent框架 | "你有没有试过让多个AI一起工作？" |
| **提示词工程** | 已成为"核心IT技能"，思维链、Few-shot等 | "你写提示词的风格是什么？长还是短？" |
| **March of Nines** | 可靠性从90%→99%→99.9%需要巨大努力 | "你有没有发现AI不够稳定的时候？" |
| **人机协作 vs 全自动** | 是辅助人还是替代人？ | "你觉得AI应该帮你做决定还是你做决定？" |

## 一、AI知识（Knowledge）

### 1.1 人工智能理解与体验 (K.UE)

#### K.UE.1 理解AI概念

**原文定义**：能够表达AI的基本概念、发展历史、阶段特点、发展现状及未来发展趋势

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 开放感知 | 你第一次接触AI是什么时候？第一印象是什么？ |
| 概念理解 | 你觉得什么是AI？你怎么跟不懂的人解释？ |
| 发展认知 | 你觉得现在AI发展到什么阶段了？跟一年前比呢？ |
| 兴趣领域 | 你有没有特别关注的AI领域？比如图像、语言、agent？ |
| 信息来源 | 你一般从哪里了解AI的新动态？播客？Twitter？公众号？ |
| 人物关注 | 你知道Karpathy、Sam Altman、Dario这些人吗？会关注他们说什么吗？ |
| 未来预期 | 你觉得AI再过两年会变成什么样？会影响你的工作吗？ |

---

#### K.UE.2 理解AI原理

**原文定义**：能够表达AI支撑技术（数据、算法、算力）和基础应用技术（CV、NLP、推理与决策等）的原理、实现过程与局限性

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 兴趣度 | 你对AI背后的技术感兴趣吗？还是只关心能用就行？ |
| 了解深度 | 你大概知道ChatGPT是怎么工作的吗？Token、模型这些词你熟吗？ |
| 学习方式 | 你有没有专门学过AI相关的东西？怎么学的？看视频？读论文？ |
| 模型认知 | 你知道GPT-4、Claude、Gemini这些有什么区别吗？怎么选？ |
| 技术边界 | 你有没有发现AI在技术上的局限？比如幻觉、上下文长度？ |
| RAG/Agent | 你听过RAG吗？Agent呢？MCP呢？知道是干什么的吗？ |
| 微调/训练 | 你有没有想过自己训练模型？或者微调？ |

---

#### K.UE.3 体验AI技术

**原文定义**：体验AI的感知、交流与行动、推理与决策能力，并能表达其局限性

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 使用频率 | 你平时用AI吗？每天用还是偶尔用？ |
| 使用场景 | 你一般用AI干什么？工作还是生活？具体是什么事？ |
| 工具选择 | 你主要用哪些AI工具？ChatGPT？Claude？Cursor？Copilot？ |
| 交流体验 | 你跟AI"聊天"的感觉怎么样？自然吗？ |
| 时长习惯 | 你一次跟AI交互多久？几分钟还是几小时？ |
| 失败体验 | 你有没有遇到AI让你很沮丧的时候？什么情况？ |
| 惊喜体验 | 有没有AI让你"卧槽"的时刻？它做了什么让你惊讶？ |
| 局限感知 | 你觉得AI什么时候特别不行？什么任务它做不了？ |
| 信任度 | 你会直接用AI的输出还是自己检查一遍？ |

## 二、AI情感（Affectivity）

### 2.1 人工智能与人类 (A.H)

#### A.H.1 AI与人类的关系

**原文定义**：能够表达AI模仿人类身体能力和认知能力，以及人机之间赋能、竞争、协同等复杂关系

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 关系隐喻 | 你觉得AI跟你是什么关系？工具？助手？同事？朋友？ |
| 拟人化 | 你有没有给AI起过名字？或者用"他/她"来称呼？ |
| 情感投射 | 你跟AI聊天会觉得开心、烦躁、感激吗？ |
| 能力对比 | 你觉得AI在哪些方面已经比你强了？哪些方面还不如你？ |
| 威胁感-职业 | 你担心过AI会取代你的工作吗？什么时候最有这种感觉？ |
| 威胁感-能力 | 你有没有觉得"我这个技能以后没用了"？ |
| 赋能感 | AI让你能做以前做不到的事吗？举个例子？ |
| 放大效应 | 你觉得AI是放大了你的优点还是暴露了你的缺点？ |
| 依赖度 | 你现在不用AI会焦虑吗？能坚持多久不用？ |
| 依赖反思 | 你觉得自己对AI的依赖是好事还是坏事？ |
| 自主性 | 你觉得什么事情应该人做决定？什么可以让AI决定？ |
| 人机边界 | 你有没有感觉AI在模糊"人做的"和"机器做的"的界限？ |
| 未来共存 | 你觉得人和AI未来会怎么相处？ |

#### A.H.2 伦理道德

**原文定义**：能够围绕AI与人的关键伦理道德问题，探究人应具备的伦理意识、态度和责任

**发散式问题池**：

| 类型 | 问题 |

| 隐私-个人 | 你会把私人的事情告诉AI吗？比如感情、健康？ |

| 署名问题 | 你会在作品里标注"AI辅助"吗？为什么？ |

| 透明度 | 你觉得别人用了AI应该告诉你吗？比如客服、医生？ |
| 价值观冲突 | 你有没有让AI做过自己都觉得有点"灰色地带"的事？ |

## 三、AI思维（Thinking）

**原文定义**：能够按照美学规律，从情感计算、艺术仿真、审美创造的角度思考设计方案

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 美学意识 | 你用AI做东西的时候会在意它好不好看吗？ |
| 用户体验 | 你会想"用户用起来感觉怎么样"吗？ |
| 细节打磨 | 你会花时间打磨细节还是只关心功能？ |
| 视觉设计 | 你用AI做过设计类的东西吗？海报、PPT、UI？ |
| 风格偏好 | 你对AI生成的风格有偏好吗？喜欢什么样的？ |
| 情感表达 | 你有没有让AI帮你做过"有情感"的东西？感动人的？ |

---

### 3.2 计算思维 (T.CT)

#### T.CT.1 分解与模块化

**原文定义**：将大问题分解成小问题，将复杂系统自顶向下划分成若干子模块

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 拆解习惯 | 你拿到复杂任务第一步做什么？ |
| 拆解方法 | 你怎么决定把任务拆成几个部分？有标准吗？ |
| 模块思维 | 你会把任务拆成独立的模块分开做吗？ |
| 分步执行 | 你会让AI一步一步做还是一次性做完？哪个效果更好？ |
| 颗粒度 | 你拆任务的时候，拆得细还是粗？ |
| 依赖管理 | 你会考虑任务之间的先后顺序吗？ |
| 失败案例 | 有没有因为没拆好导致AI做不好的经历？ |

---

#### T.CT.2 抽象与建模

**原文定义**：运用计算机科学思想方法，通过问题抽象来形成模型化的问题解决方案

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 问题抽象 | 你怎么把业务问题"翻译"成AI能理解的形式？ |
| 提示词结构 | 你的提示词有固定结构吗？比如角色+任务+格式？ |
| 模式识别 | 你有没有发现"这个问题跟之前那个类似"？ |
| 框架思维 | 你有没有自己总结的AI使用框架或者方法论？ |
| 模板积累 | 你有没有自己积累的提示词模板？ |
| 元认知 | 你会反思"我是怎么用AI的"吗？ |
| 抽象能力来源 | 你觉得这种"抽象问题"的能力是天生的还是练出来的？ |

---

#### T.CT.3 数据与标注

**原文定义**：对文本、视频、图像等原始数据添加元数据进行标注，形成训练数据集

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 数据意识 | 你用AI的时候会想到"数据"这个概念吗？ |
| 数据整理 | 你有没有自己整理过数据给AI用？比如清洗、格式化？ |
| 知识库 | 你有没有给AI建过"知识库"或者"上下文"？ |
| RAG使用 | 你用过RAG吗？让AI读你自己的文档？ |
| 标注经验 | 你做过数据标注吗？觉得难吗？ |

---

#### T.CT.4 训练与模拟

**原文定义**：将标记好的数据用于训练模型，通过模拟、仿真验证效果

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 微调经验 | 你试过微调模型吗？Fine-tune？ |
| 训练认知 | 你知道训练模型需要什么吗？ |
| 模拟测试 | 你会先在"测试环境"里试AI效果吗？ |
| 效果预期 | 你怎么预估AI能做到什么程度？ |

---

#### T.CT.5 部署与推理

**原文定义**：将训练好的模型部署到设备中，接收新数据进行推理与预测

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 部署经验 | 你把AI应用部署上线过吗？ |
| 集成方式 | 你怎么把AI集成到工作流里？API？插件？ |
| 自动化 | 你有没有让AI自动运行的场景？不需要你手动触发？ |
| 监控维护 | 上线后你会监控AI的效果吗？ |

---

#### T.CT.6 优化与迭代

**原文定义**：持续反思模型训练方案的不足，逐步求精和优化完善

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 效果评估 | 你怎么知道AI做得好不好？有标准吗？ |
| 满意度 | 什么情况下你会觉得"这个AI方案够好了"？ |
| 改进循环 | 你改过几轮？每轮改什么？ |
| 退出标准 | 你什么时候决定"不改了"？ |
| 持续优化 | 上线后你还会继续改吗？还是一劳永逸？ |
| 版本管理 | 你会保存不同版本的提示词或者方案吗？ |
| A/B测试 | 你有没有比较过两种AI方案哪个更好？ |

---

#### T.CT.7 复用与迁移

**原文定义**：利用已有问题解决方案，迁移运用于解决其他问题

**发散式问题池**：

| 类型 | 问题 |
|------|------|
| 经验复用 | 你有没有把一个项目的AI经验用到另一个？ |
| 提示词库 | 你有没有自己积累的提示词模板库？ |
| 工作流复用 | 你的AI工作流可以复制粘贴到别的任务吗？ |
| 知识迁移 | 你觉得AI使用能力可以迁移到不同领域吗？ |
| 教学传递 | 你把自己的AI方法教给别人过吗？ |
| 行业通用性 | 你觉得你的AI技能换个行业还能用吗？ |
| 学习曲线 | 新人学你的AI方法难吗？ |
