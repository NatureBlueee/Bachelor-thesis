# Task-0003: 访谈提纲设计

> **状态**: ✅ 已完成（2026-01-02更新）
> **目标**: 基于Task-0001/Task-0002的~275个探针，设计RQ导向的访谈提纲
> **迭代**: 根据Cambia访谈反思进行技术调整

---

## 🎯 核心锚点（访谈时心里记住这个）

> **核心问题**：AI作为一种素养，怎么改变上下级关系？

### 三维度抽象定义（记住定义，而非具体问题）

| 维度 | 核心定义 | 我在听什么 |
|------|----------|------------|
| **AI知识** | 他知道AI是什么、怎么工作、怎么用 | 术语熟悉度、学习来源、技术深度 |
| **AI情感** | 他对AI的感觉（工具 vs 朋友 vs 威胁） | 拟人化程度、依赖/焦虑、价值观 |
| **AI思维** | 他怎么用AI解决问题、设计流程 | 拆解习惯、迭代方式、框架意识 |

---

## 设计原则

1. **记住定义，而非背问题**：抽象定义在心里，具体问题即兴生成
2. **按RQ组织**，而非按变量
3. **"朋友房间"模式**：自然对话，不是问答
4. **灵活适应**：根据受访者反应选择追问方向
5. **顺着聊**：允许话题自然延伸，不要生硬跳转

---

## 访谈结构总览

| 阶段 | 时长 | 内容 | 核心RQ |
|------|------|------|--------|
| Opening | 5min | 建立关系、背景介绍 | - |
| Block 1 | 15min | AI素养形成与感知 | RQ1 |
| Block 2 | 20min | 向上影响行为与机制 | RQ2 |
| Block 3 | 10min | 上级开放性的角色 | RQ3 |
| Closing | 5min | 补充、职业展望 | - |
| **总计** | **~55min** | | |

## Opening（5分钟）

### 目标：建立信任、了解背景

**核心问题**：
> "先简单介绍一下你自己吧——你现在做什么工作，多久了，上级是个什么样的人？"

**追问选项**：
- 你们团队有多少人？
- 你跟上级互动多吗？
- 你觉得你们的关系怎么样？

---

## Block 1: AI素养形成与感知（15分钟）

### 对应：RQ1 - How do Generation Z employees perceive their AI literacy and its development?

### 理论来源：钟柏昌 AI素养三维框架 (Task-0001)

---

### 1.1 AI接触与使用（K.UE.3 体验）

**核心问题**：
> "聊聊你跟AI的关系吧——你平时用AI吗？怎么用的？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 频率 | 每天用还是偶尔？ |
| 工具 | 主要用什么？ChatGPT？Claude？Cursor？ |
| 场景 | 工作用还是生活用？具体做什么？ |
| 时长 | 一次跟AI交互多久？ |
| 惊喜 | 有没有AI让你惊喜的时刻？ |
| 沮丧 | 有没有AI让你沮丧的时候？ |

---

### 1.2 AI知识与理解（K.UE.1-2 概念与原理）

**核心问题**：
> "你对AI技术本身感兴趣吗？还是只关心能用？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 学习方式 | 你是怎么学AI的？看视频？自己摸索？ |
| 技术深度 | 你大概知道ChatGPT是怎么工作的吗？ |
| 术语熟悉 | Token、RAG、Agent这些词你熟吗？ |
| 模型区分 | GPT-4、Claude、Gemini有什么区别你知道吗？ |
| 信息来源 | 你一般从哪里了解AI新动态？ |
| 人物关注 | 你知道Karpathy、Sam Altman这些人吗？ |

---

### 1.3 AI态度与情感（A.H.1 人机关系）

**核心问题**：
> "你觉得AI跟你是什么关系？工具？助手？还是别的？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 依赖 | 你现在不用AI会焦虑吗？ |
| 威胁 | 你担心过AI取代你吗？ |
| 赋能 | AI让你能做以前做不到的事吗？ |
| 边界 | 什么事应该人做，什么可以给AI？ |
| 伦理 | 你用AI的时候有没有想过伦理问题？ |

---

### 1.4 AI思维方式（T.CT.1-2 分解与抽象）

**核心问题**：
> "你用AI处理复杂任务的时候，一般怎么做？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 拆解 | 你会先把任务拆开吗？ |
| 提示词 | 你的提示词一般多长？有固定结构吗？ |
| 迭代 | 不满意的时候你怎么办？ |
| 框架 | 你有没有自己总结的AI使用方法？ |

---

### 1.5 与他人比较（过渡到RQ2）

**核心问题**：
> "你觉得你在AI方面，比周围的人怎么样？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 同事比较 | 跟同事比呢？ |
| 上级比较 | 跟你上级比呢？ |
| 差距感知 | 差距大吗？体现在哪里？ |
| 优势感知 | 你有没有"在这方面我比上级更懂"的感觉？ |

## Block 2: 向上影响行为与机制（20分钟）

### 对应：RQ2 - How does AI literacy influence upward influence behavior?

### 理论来源：RDT + 领导身份建构 (Task-0002)

---

### 2.1 资源优势感知（RDT 2.4）

**核心问题**：
> "你刚才说在AI方面比上级强，这种感觉什么时候最明显？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 具体情境 | 能举个例子吗？ |
| 信息差 | 有没有你知道但上级不知道的AI信息？ |
| 工具专长 | 有没有什么工具只有你会用？ |
| 依赖感知 | 你觉得上级在AI的事情上依赖你吗？ |

---

### 2.2 能力展示/Claiming（领导身份建构 3.1）

**核心问题**：
> "你会主动让上级知道你在AI方面比较强吗？怎么展示的？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 展示场景 | 在什么场合展示？会议？私聊？工作成果？ |
| 主动vs被动 | 是你主动展示还是别人问你才说？ |
| 展示动机 | 你为什么展示？想获得什么？ |
| 风险考量 | 展示有风险吗？比如被当成"爱表现"？ |
| 策略 | 你会考虑"在谁面前展示"吗？ |

---

### 2.3 向上影响经历（核心！）

**核心问题**：
> "你有没有用自己的AI知识影响过上级的决定或者做事方式？"

**追问选项（如果有）**：
| 方向 | 问题 |
|------|------|
| 具体情况 | 能详细说说那次经历吗？ |
| 你做了什么 | 你是怎么提出建议的？ |
| 上级反应 | 上级当时怎么反应？ |
| 结果 | 最后怎么样了？建议被采纳了吗？ |
| 感受 | 你当时什么感觉？ |
| 多次经历 | 这样的事情发生过几次？ |

**追问选项（如果没有）**：
| 方向 | 问题 |
|------|------|
| 原因 | 为什么没有提过？ |
| 想法 | 有没有想过要提但没提的时候？ |
| 顾虑 | 什么顾虑让你没提？ |
| 假设 | 如果你提了，你觉得上级会怎么反应？ |

---

### 2.4 被认可/Granting（领导身份建构 3.2）

**核心问题**：
> "你觉得上级认可你的AI能力吗？怎么表现的？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 言语认可 | 上级说过"你AI很厉害"之类的话吗？ |
| 任务分配 | 上级会把AI相关的事交给你做吗？ |
| 咨询行为 | 上级遇到AI问题会找你吗？ |
| 公开认可 | 上级在别人面前提起过你的AI能力吗？ |
| 角色赋予 | 你在团队里算"AI专家"吗？ |

---

### 2.5 影响力循环（领导身份建构 3.3）

**核心问题**：
> "你提了建议被接受/拒绝之后，下次还会提吗？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 正向循环 | 被认可后你更敢提建议了吗？ |
| 负向循环 | 有没有因为没被认可就不想再主动的时候？ |
| 策略调整 | 被拒绝后你的做法会变吗？ |
| 信心变化 | 这些经历对你的信心有影响吗？ |

## Block 3: 上级开放性的角色（10分钟）

### 对应：RQ3 - What role does supervisor openness play?

### 理论来源：Detert & Burris (Task-0002)

---

### 3.1 倾听与兴趣（Listens + Interested）

**核心问题**：
> "你跟上级谈AI的事情时，他一般什么反应？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 倾听行为 | 他会认真听吗？还是边听边干别的？ |
| 追问深度 | 你说完后他会追问细节吗？ |
| 兴趣表现 | 他对AI话题有兴趣吗？ |
| 请教行为 | 他有没有主动问过你"这个怎么弄"？ |
| 学习意愿 | 他有没有表示想学AI？ |

---

### 3.2 公正考虑与行动（Fair Consideration + Takes Action）

**核心问题**：
> "你的AI建议有被采纳过吗？没被采纳的时候上级怎么说？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 采纳经历 | 哪些建议被采纳了？ |
| 拒绝解释 | 不采纳的时候他会解释原因吗？ |
| 公平感 | 你觉得他评估你的建议公平吗？ |
| 行动速度 | 从你提建议到他行动，要多久？ |
| 资源支持 | 他会给你资源/时间去实施吗？ |

---

### 3.3 心理安全（Psychological Safety）

**核心问题**：
> "你觉得向上级提AI建议有风险吗？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 风险感知 | 什么风险？ |
| 最坏情况 | 你担心提了建议后会怎样？ |
| 出错代价 | 如果你的建议出了问题，你会被责怪吗？ |
| 尝试空间 | 上级允许你"试错"吗？ |
| 自我审查 | 有没有话你想说但选择不说？ |

---

### 3.4 上级开放性的影响（核心）

**核心问题**：
> "上级的态度会影响你提建议的意愿吗？怎么影响的？"

**追问选项**：
| 方向 | 问题 |
|------|------|
| 正向影响 | 什么样的上级态度让你更愿意说？ |
| 负向影响 | 什么样的态度让你不想说？ |
| 理想上级 | 你理想中对AI开放的上级是什么样的？ |
| 代际因素 | 你觉得上级的年龄影响他的开放性吗？ |

## "朋友房间"模式Checklist

### 访谈前
- [ ] 准备好录音设备（线上可直接录屏）
- [ ] 熟悉受访者背景（如有）
- [ ] **记住三维度定义，不要背问题清单**
- [ ] 判断受访者类型（爱表达 vs 不爱表达）

### 访谈中
- [ ] 不要读问题，用自然的对话
- [ ] **识别受访者节奏**：爱表达→少问多听；不爱表达→一点点引导
- [ ] 允许话题自然延伸，**不要生硬跳到下一个Block**
- [ ] 如果受访者自己聊到了后面的话题，顺着聊
- [ ] 用"然后呢"、"具体是..."、"能举个例子吗"推进
- [ ] **追问来自兴趣，而非覆盖清单**

### 数据记录
- [ ] 记下关键词和时间戳
- [ ] 标注特别有价值的片段
- [ ] **访谈后立即写备忘录**（见模板）

## 访谈技术反思（基于Cambia访谈 2026-01-02）

### 核心教训

| 问题 | 反思 | 调整 |
|------|------|------|
| 预设问题太死板 | 对于思考丰富的人，应更自由地顺着话题走 | 记住定义，即兴生成问题 |
| 从一个点跳到另一个点 | 会让人感到"刚才没延续下去" | 保持对话自然流动 |
| 为了覆盖点而生硬追问 | 打断了有价值的深入讨论 | 追问来自真正的兴趣 |

### 不同受访者策略（关键！）

| 受访者类型 | 特征 | 策略 |
|------------|------|------|
| **思考丰富/爱表达** | 主动分享、有自己的框架、话多 | 少问具体问题；多用"然后呢"；让他讲故事；顺着他的逻辑走 |
| **思考较少/不爱表达** | 回答简短、需要引导 | 可用预设问题一点点推进；问事实层面的；不追求深度 |

### 线上访谈优势

- ✅ 对方在自己环境更放松
- ✅ 录音转录方便
- ✅ 时间灵活，易安排
- ✅ 对于爱表达的人，线上也能充分展开
- ✅ 不爱表达的人可能更快完成（效率高）
