## 4. 项目简介与核心创意 (200字以内)

> 一个多Agent协作的学术研究助手系统。解决的问题：**AI写作容易"幻觉"、缺乏批判性思维、修改不可追溯**。
>
> 创新点：
> 1. **负面约束设计**：每个Agent被明确告知"不能做什么"（如文献Agent不能编造引用）
> 2. **对抗性协作**：Critical Thinker Agent专门质疑其他Agent的输出
> 3. **PR驱动变更**：所有论文修改必须通过PR Manager审核，可追溯
> 4. **真实验证**：已用于一篇本科毕业论文的实际写作
>
> 场景：用户提出修改需求 → Facilitator协调 → 各专家Agent讨论 → 达成共识 → PR Manager生成变更单。

---

## 5. 技术方案简述

**计划创建6个Agent：**

| Agent | 角色 | 协作方式 |
|-------|------|----------|
| Literature Agent | 文献检索与管理 | 被动响应查询，主动补充引用 |
| Critical Thinker | 批判性质疑 | 对其他Agent输出进行审查 |
| Writing Assistant | 格式润色 | 检查APA格式、语言规范 |
| Method Expert | 方法论顾问 | 提供研究设计建议 |
| Consensus Facilitator | 协调者 | 组织讨论流程，推动共识 |
| PR Manager | 变更管理 | 生成、追踪、合并修改请求 |

**使用的OpenAgents特性：**
- **messaging Mod**：Agent间通过Channel通信
- **forum Mod**：用于结构化讨论（如方法论辩论）
- **wiki Mod**：共享知识库（文献索引）
- **gRPC协议**：Agent与Network通信
- **事件系统**：PR状态变更触发通知

## 6. 预期实现的功能清单

1. **Literature Agent**：接收研究问题，从本地文献库检索相关PDF，生成结构化摘要
2. **Critical Thinker**：监听讨论频道，对论点提出质疑，要求提供证据
3. **Writing Assistant**：检查论文格式，标记APA引用错误
4. **Method Expert**：回答方法论问题，推荐研究设计
5. **Consensus Facilitator**：当讨论超过3轮无结论时，总结各方观点并推动投票
6. **PR Manager**：将达成共识的修改生成PR文件，追踪状态（planning→approved→merged）
7. **自定义 Mod**：Document Mod，允许Agent访问本地文件系统（Reference/、PR/、Target/）

## 8. 其他需要说明的情况

本项目源于一个真实的本科毕业论文写作场景。项目核心理念是"通过负面约束防止AI幻觉"——与其告诉AI"要做什么"，不如明确告诉它"不能做什么"。

目前已完成：
- 6个Agent的Prompt设计与YAML配置
- 本地OpenAgents Network环境搭建
- 论文PR管理系统原型（非Agent版本已在使用）

参赛目标：将现有的单Agent协作升级为真正的多Agent网络，验证"对抗性协作"能否提升学术写作质量。
