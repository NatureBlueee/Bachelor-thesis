## 项目定位

**不是**"论文写作工具"。

**是**：长期、复杂、迭代式创造性工作的人机协作系统。

核心特征：
- 资源需求广泛，来源要求高（不是随便搜一搜）
- 长期工作（像开发代码一样开发文档）
- 内容不限定（论文、书籍、研究、策略、任何复杂决策文档）

## 差异化亮点

### 1. 负面约束设计（`.agent/rules/`）

> 在研究的60+项目中，只有 data-to-paper 和 Cursor 有类似概念。

大多数工具告诉AI"应该做什么"，我们告诉AI"**绝对不能做什么**"：

```markdown
❌ 禁止直接编辑论文 → 必须通过 /create_pr
❌ 禁止凭空捏造文献 → 论据必须来自 Reference/
❌ 禁止只给结论 → 必须要求颗粒度
```

### 2. PR驱动的知识迭代

不是"AI帮你写"，而是"AI和你协作审核"。每次修改都是一个可追溯的PR。

### 3. Consensus驱动的结构化思考

强制走完思考流程：问题定义 → 假设 → 证据 → 推理 → 反驳 → 决策

## 6个核心Agent设计

| Agent | 职责 | 对应模块 |
|-------|------|----------|
| **Literature Agent** | 文献检索、推荐、PDF转换 | Reference/ |
| **Consensus Facilitator** | 组织结构化讨论、确保流程完整 | Consensus/ |
| **Critical Thinker** | 质疑论证、发现逻辑漏洞 | 魔鬼代言人 |
| **Writing Assistant** | 辅助写作、格式检查 | Target/ |
| **Method Expert** | 方法论建议、研究设计 | Methodology |
| **PR Manager** | 变更管理、召集review | PR/ |

## 演示场景设计

**场景：用户想修改论文的研究方法章节**

```
Step 1: 用户发起
   用户: "我想把定性研究改成混合方法"

Step 2: Facilitator组织讨论
   Facilitator: "这是个重要决策，让我召集专家讨论"
   → 创建Consensus讨论线程

Step 3: 多Agent辩论
   Method Expert: "混合方法可以增强说服力，但需要额外时间"
   Critical Thinker: "你确定不是担心导师觉得定性'不够科学'？"
   Literature Agent: "我找到3篇类似研究都用定性。"

Step 4: 形成决策
   Facilitator: "建议保持定性，但在讨论章节对比定量研究。"
   用户: "好，接受"

Step 5: PR Manager执行
   PR Manager: "我创建了PR-0015。"
   Writing Assistant: "草稿已写好，请review。"
   → 合并到Draft.md
```

**整个流程：5分钟，6个Agent协作完成！**
