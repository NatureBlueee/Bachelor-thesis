# 论文项目启动检查点

> [!IMPORTANT]
> 这是一个**强制性**的上下文加载流程。每个新对话开始时执行。
> **顺序很重要**：最后读Draft.md，因为论文正文需要在工作记忆中最清晰。

## 第一步：加载项目状态

// turbo
1. **进度共识**: `Consensus/_CONTEXT.md`
   - 了解当前阶段、已完成什么、下一步是什么

## 第二步：检查活动PR

// turbo
2. 列出 `PR/` 目录内容，检查是否有状态为 `🟡 讨论中` 的PR
   - 如果有，读取其内容，了解当前正在进行的修改

## 第三步：加载AI笔记

// turbo
3. **AI笔记**: `MEMORY.md`
   - 关键洞见、用户偏好、待解决问题、流程规则

## 第四步：理解论文核心（最后读，印象最深）

// turbo
4. **读取 `Target/Draft.md` 的核心章节**：
   
   | 行范围 | 内容 | 为什么重要 |
   |--------|------|------------|
   | 1-60 | 中英文摘要 | 快速理解研究主题、方法、贡献 |
   | 178-205 | 研究目标与意义 | 理解我们要回答什么问题 |
   | 340-400 | 理论框架与命题P1-P4 | 这是论文的核心逻辑 |
   
   **核心理解检查**（读完后应能回答）：
   - [ ] 这篇论文研究什么？（AI素养→感知资源优势→向上影响行为）
   - [ ] 主要理论是什么？（资源依赖理论）
   - [ ] 四个命题分别是什么？
   - [ ] 调节变量是什么？（上级开放性）

## 第五步：输出状态报告

5. 向用户输出简洁的状态报告：

```markdown
## 📍 论文项目状态

**研究主题**: AI素养对Z世代员工向上影响行为的影响
**当前阶段**: [从_CONTEXT.md获取]

### ✅ 最近完成
- [1-2项]

### 🚧 进行中 / 活动PR
- [当前任务]

### 🎯 待办优先级
- [最重要的1-2项]

---
准备就绪。请问今天要做什么？
```

---

## 使用方式

用户只需输入：
```
/start
```

或者类似表达："继续论文"、"开始工作"、"我们继续"

---

## 注意事项

- 如果对话已经很长，中途可以再次执行 `/start` 刷新上下文
- 遇到复杂任务时，主动提醒用户"我回去看一下workflow文件"
- 合并PR等关键操作前，确认已读取相关workflow
- **关键规则**（来自 thesis-project.md）：
  - 不直接编辑Draft.md，必须通过 `/create_pr`
  - 不凭空捏造文献，论据必须来自 `Reference/`
  - 修改后需更新 `_CONTEXT.md`
