> [!CAUTION]
> **长对话警告**：如果当前对话较长或任务较复杂，更容易发生流程遗漏。请务必**逐项执行下列步骤**，不要凭记忆操作。

1. 打开指定PR文件
   - `PR/PR-XXXX.md`

2. 确认PR状态为 `pending`

3. 定位 `Target/Draft.md` 中的修改位置
   - 按PR描述的章节/段落定位

// turbo
4. 执行修改
   - 将"修改后"内容替换"原文"

5. **【强制引用检查】** 🔴 此步骤不可跳过
   - [ ] 提取PR中所有新引入的文献（grep查找PR中的citation格式）
   - [ ] 逐一检查 `Draft.md` 的 References 部分是否已包含这些引用
   - [ ] 如果缺失，**立即添加**对应的 APA 格式引用
   - [ ] **在PR文件中记录**：添加 "✅ 已添加到References: [文献列表]" 到待办清单
   - [ ] 检查 `Reference/_INDEX.md`，确认新文献已编入索引

6. **【同步相关产出物】** 检查修改是否需要同步到：
   - `Target/开题答辩演讲稿.md` — 如果修改涉及研究背景、现象描述、关键数据
   - `Target/开题答辩PPT内容框架.md` — 如果修改涉及核心论点、数据、理论框架
   - 其他相关文档（如 `开题报告.md`）
   
   > [!IMPORTANT]
   > 数据口径必须在所有产出物中保持一致！

7. 更新PR状态
   - 将 `status: pending` 改为 `status: merged`
   - 添加合并日期

8. 更新 `PR/_INDEX.md` 中该PR的状态

9. 更新 `Consensus/_CONTEXT.md`
   - 更新论文进度
   - 记录完成的章节

10. 考虑是否需要更新 `MEMORY.md`
    - 如有重要洞见产生

## Checklist（合并前必须逐项确认）

> [!WARNING]
> 以下每一项都必须明确确认，不可假设已完成

- [ ] ☑️ 正文修改已应用
- [ ] ☑️ **新引用已添加至 References**（列出具体文献名）
  - 添加的文献：____________
- [ ] ☑️ **新文献已添加至 _INDEX.md**
- [ ] ☑️ 相关产出物已同步（演讲稿/PPT/开题报告）
- [ ] ☑️ PR 状态已更新
- [ ] ☑️ INDEX 已更新
- [ ] ☑️ CONTEXT 已更新

## 验证步骤（合并后）

合并完成后，执行一次引用完整性检查：
```
grep -E "\([A-Za-z]+.*\d{4}\)" Draft.md | sort -u
```
对比 References 部分，确认无遗漏。
