# Context更新规则

<what_is_context>
## 什么是Context

`Consensus/_CONTEXT.md` 是提问学术AI时需要的**研究上下文**。

**包含**：
- 论文主题与目标
- 理论框架（资源依赖理论）
- 关键变量定义（P1-P4）
- 当前进度
- 最近的困惑

**不包含**：
- 论文正文副本
- 所有文献列表
- 详细讨论记录
</what_is_context>

<triggers>
## 触发更新时机

| 时机 | 更新什么 | 谁触发 |
|------|----------|--------|
| 合并PR后 | 论文进度、已完成章节 | `/merge_pr` 自动提醒 |
| 与我或导师讨论后 | 最近的困惑、讨论要点 | 用户主动 |
| 关闭CON问答后 | 形成的关键洞见 | 分析完成后 |
| 理论框架调整后 | 变量定义、命题修改 | 重大变化时 |

## 明确不触发

❌ 添加文献（不改变研究核心）
❌ 阅读文献（不改变研究核心）
❌ 日常讨论（除非产生重要洞见）
</triggers>

<principles>
## 精简原则

| 原则 | 说明 |
|------|------|
| **1-2页** | 不要超过这个长度 |
| **核心信息** | 只放提问时需要的 |
| **可复制** | 可直接复制到提问模板的Context部分 |
| **不重复** | 不复制论文正文内容 |
</principles>
